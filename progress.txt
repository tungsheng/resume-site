API NOTES (verified from installed satteri@0.9.1 dist types — kept for ref)

- Plugin helpers: defineMdastPlugin / defineHastPlugin from "satteri".
  Register via satteri({ mdastPlugins:[...], hastPlugins:[...] }).
- Admonition (#16): defineMdastPlugin({ name, blockquote(node, ctx) }).
- Blog images (#17): defineHastPlugin({ name,
    element: { filter:['img','p'], visit(node, ctx) } }).

  CONTEXT, not in place: ctx.replaceNode(node, newNode),
  ctx.setProperty(node,'data',{ hName, hProperties }), ctx.prependChild(...),
  ctx.removeNode(...), ctx.insertChildAt(...). data.hName/hProperties ARE
  honored (so the callout div + classes + title-prepend mechanism carries
  over). The current transformAdmonitions/transformBlogImages mutate trees
  directly, so the port rewrites the MECHANISM; the marker-detect / tone-map /
  figure-wrap LOGIC and the unit-test expectations carry over.
- features.gfm DEFAULTS TRUE (ADR said "opt-in" — reality: on by default);
  features.smartPunctuation defaults FALSE (must enable, done in #15).
