# Blog navigation: governed tag registry, static tag pages, Pagefind search

Status: accepted

As the Post count grows, Tags (not Categories, and not a Series concept) become the reader's primary navigation axis — every existing Post is `category: Inference`, so the Category enum does no organizational work, while the tag vocabulary is where the real topology lives. Three decisions follow:

1. **Governed tag registry.** Tags are drawn from a registry in code (canonical kebab-case slug + display label) validated by the frontmatter schema, so an unregistered or misspelled tag fails the build. The existing corpus forked its vocabulary within weeks (`kv-cache` vs `KV cache`, `moe` + `mixture-of-experts` on one Post) — with tag URLs, each fork is a duplicate half-empty page. The registry is open for extension while writing, unlike the closed Category enum. A one-time cleanup merges the existing forks.
2. **Static tag pages** (`/blog/tags/[slug]` + a `/blog/tags` overview) are the canonical tag surface — the no-JS baseline ADR-0008 requires. No minimum-post-count gate: a gate makes tag pages flicker in and out of existence as counts change. There is deliberately **no standalone client-side filter island** on the blog index — tag filtering rides inside the search UI instead (decision 3), so a separate filter island would be obsolete on arrival.
3. **Pagefind, blog-only.** Search is a build-time static index over the emitted `dist/` HTML (Pagefind after `astro build`), shipped as a lazy-loaded island per ADR-0008 — no backend, no external service. Draft exclusion is free by construction: Outline/Drafting Posts never reach `dist/` (ADR-0003 §8), so they cannot be indexed. Tag and Category become `data-pagefind-filter` facets. Scope is the Blog only (`data-pagefind-body` on the post layout); site-wide indexing is additive later. Rejected: hand-rolled MiniSearch/Fuse index (maintaining index format + UI unjustified at this scale), external engines like Algolia (external dependency for a personal blog), and deferring (integration is cheap; the corpus is growing ~2 Posts/week).
